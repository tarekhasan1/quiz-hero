1. What is the difference between map, filter, forEach and find methods?

map(): map function receives a callback function as a parameter and will apply the code on each element and returns a new array.

filter(): filter method receives a callback function as a parameter and it runs the function for each element in the array and return the new array which satisfies the condition applied.

forEach():forEach function receives a callback function as an argument and it applies the same code to every element and it will not return anything. the return value of forEach function is undefined.

find():find method receives a callback function as a parameter and it runs the function for each element in the array and return only the first element of the array which satisfy the condition applied.

2. What is the difference between local storage, session storage and cookies?

There are three mechanism for storing data on the client in javascript, they are-
local storage,
session storage and
cookies.

local storage allows for larger amounts of data to be stored, but the data is not deleted when the browser is closed. Local storage is useful for storing data that the user will need to access later, such as offline data.

session storage is similar to cookies, but the data is only stored for the current session. This means that the data will be deleted when the user closes the browser. Session storage is useful for storing data that is sensitive, such as login credentials.

cookies are the oldest and most well-known mechanism. They are simple to use and well supported by browsers. However, they are limited to 4KB of data and are often used to store data that is not sensitive, such as user preferences.
